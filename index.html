Краткий гайд по работе с репозиторием

Введение

Работать с репозиторием можно разными способами. Толстый клиент GitHub - самый простой. Он потребует установки приложения (примерно 45Мб) и регистрации (нужен только email). Просматривать файлы удобно и через Web-интерфейс самого GitHub, однако для полноценной работы он не так удобен.

В данном руководстве рассмотрим настройку и работу через толстый клиент.

Работа через толстый клиент

Установка клиента

Ссылка на скачивание онлайн-инсталлятора:

  1. Windows: https://windows.github.com/
  2. Mac: https://mac.github.com/

Качаем, ставим. А в это время:

Регистрация на GitHub (если ещё нет аккаунта)

  1. Заходим на https://github.com
  2. Жмём sign up
  3. Нужно ввести имя пользователя (логин) и email для регистрации, а также пароль
  4. После подтверждения данных - выбрать план Free (по-умолчанию) - бесплатный. Все функции GitHub будут доступны.
  5. После успешной регистрации и логина, главная страница github.com выглядит так:

Вступление в группу

  1. Просим в чатике скайпа (course BD and PA) добавить ваш логин в группу
  2. На почту придёт инвайт, в нём необходимо перейти по ссылке, или зайти на страничку группы: https://github.com/course-bd-and-pa-bsuir
  3. Перейдя на страницу группы необходимо подтвердить своё участие

Скачивание репозитория

  1. Запустить и залогиниться в толстом клиенте
  2. При первом запуске необходимо настроить свои имя и email, которые будут публично видны в коммитах репозитория
  3. Зайти на страницу репозитория: https://github.com/course-bd-and-pa-bsuir/r-statics и нажать справа внизу кнопку "Clone in desktop"
  4. Браузер может спросить о запуске ссылки через толстый клиент, фокус должен перейти на его окно.
  5. Необходимо выбрать папку для репозитория. По умолчанию, на Windows это %USER_HOME%/Documents/GitHub
  6. Во время клонирования будет отображаться прогресс, по окончании процесса репозиторий будет доступен на файловой системе для просмотра и изменения

Работа с репозиторием

  1. Работать с репозиторием надо на файловой системе. Толстый клиент нужен только для синхронизации с GitHub - заливки и подтягивания изменений.
  2. Создайте в корне репозитория отдельную папочку для себя и храните свои файлы в ней
  3. После добавления/удаления/изменения файлов их необходимо коммитить в толстом клиенте. Добавляйте словесное описание к коммитам, отражающее суть сделанных изменений.
  4. После коммита (одного или нескольких) изменения нужно синхронизировать. Для этого в толстом клиенте есть кнопочка "Sync".
  5. Также можно (и нужно) выполнять периодическую синхронизацию для получения данных с GitHub.
